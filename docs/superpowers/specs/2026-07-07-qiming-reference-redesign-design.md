# 启名 Qiming · 参考图完整还原设计文档

## 1. 目标

参考用户提供的「沐阳起名」风格截图，将现有启名应用升级为三列布局、信息密度更高、交互更完整的宝宝取名界面。一次到位补齐顶部导航、收藏/历史/设置、八字命盘可视化、名字对比、筛选排序等能力。

## 2. 页面结构

```
┌──────────────────────────────────────────────────────────────┐
│  AppHeader（Logo + 标题 + 使用指南 / 我的收藏 / 历史记录 / 设置）  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  / (首页)        落地页，介绍五大维度，引导进入 /generate        │
│  /generate       三列取名流程（核心页面）                       │
│  /saved          我的收藏                                      │
│  /history        历史记录                                      │
│  /guide          使用指南                                      │
│  /settings       设置                                          │
│  /name/[id]      名字详情页（保留，与右侧面板内容一致）          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 3. /generate 三列布局

### 3.1 左列：宝宝信息表单

- **基本信息**
  - 姓氏（必填，1-2 字）
  - 性别倾向：男 / 女 / 中性（按钮式单选）
  - 出生时间：公历 / 农历 切换，日期时间选择器
  - 出生地：中国省 / 市 / 区 三级级联选择器
- **名字设置**
  - 名字字数：双字名 / 单字名
  - 辈分字（选填）+ 位置（首字 / 末字）
  - 喜欢的字（多选输入）
  - 避讳的字（多选输入）
- **风格偏好**
  - 古典文雅 / 温润平和 / 明朗大气 / 诗词典故 / 现代简约
  - 每条风格用滑块控制强度（0-100），前端以权重形式传给 LLM
- **更多筛选**
  - 避开生僻字 / 避开多音字 / 避开谐音不雅 / 笔画不宜过多（≤20 画） / 仅看高分名字
- **操作**
  - 清空按钮
  - 「生成好名」主按钮

### 3.2 中列：推荐名字列表

- 头部：「为您推荐的名字 共 N 个好名」+ 排序下拉（综合推荐 / 五行补益优先 / 音律优先 / 寓意优先）+ 筛选 + 换一批
- 表头：推荐 / 姓名 / 综合评分 / 五行补益 / 寓意摘要 / 谐音风险 / 收藏
- 每行：
  - 序号
  - 多选框（加入对比）
  - 收藏心形
  - 姓名 + 拼音
  - 综合评分（≥90 绿色 / 75-89 琥珀色 / <75 灰色）
  - 五行补益标签（如「补木、火」）
  - 寓意摘要（一句话）
  - 谐音风险（风险低 / 中 / 高）
  - 点击整行选中，右侧面板显示详情
- 底部分页器

### 3.3 右列：名字解析面板

- 顶部：名字 + 拼音 + 综合评分 + 已收藏状态
- Tab 导航：八字五行 / 音律分析 / 字形结构 / 寓意解析 / 出处典故 / 避讳提醒
- **八字五行 Tab**
  - 五行分布条形图（木/火/土/金/水 数量）
  - 喜用神说明
  - 八字命盘：年柱 / 月柱 / 日柱 / 时柱，每柱显示天干、地支、五行
  - 命理简析文本
- **音律分析 Tab**：声调、平仄、韵律、读感说明
- **字形结构 Tab**：笔画数、偏旁、结构美感
- **寓意解析 Tab**：整体寓意
- **出处典故 Tab**：诗句、诗题、作者、朝代、关联说明
- **避讳提醒 Tab**：谐音、生僻字、多音字检查结果
- **底部**
  - 对比名字区（最多 3 个，可添加/移除）
  - 综合评价评分条（八字五行 / 音律音调 / 字形结构 / 寓意内涵 / 避讳风险 / 风格匹配）
  - 「查看完整解析报告」按钮（弹窗/新页）

## 4. 新增页面

### 4.1 /saved（我的收藏）

- 展示所有收藏名字卡片
- 支持取消收藏、批量选择、加入对比、生成报告
- 空状态引导去取名

### 4.2 /history（历史记录）

- 按时间倒序展示每次生成记录
- 每条记录展示：时间、姓氏、性别、生成数量、前几个名字
- 支持「重新加载参数」回到 /generate 并自动填充
- 支持删除单条 / 清空全部

### 4.3 /guide（使用指南）

- 解释五大维度：八字五行、诗词典故、音律字形、算法推荐、避讳查重
- 图文说明如何填写表单和解读结果

### 4.4 /settings（设置）

- LLM provider / model / API key（仅前端展示当前环境变量配置，敏感信息不在前端修改）
- 数据管理：导出收藏/历史、清空收藏/历史
- 主题切换（可选，首期保持浅色）

## 5. 数据流与状态

### 5.1 后端 API 调整

- `/api/generate`
  - prompt 要求 LLM 返回 `baziSummary` 时同时返回结构化数据：
    ```ts
    baziPillars?: {
      year: { gan: string; zhi: string; wuxing: string };
      month: { gan: string; zhi: string; wuxing: string };
      day: { gan: string; zhi: string; wuxing: string };
      hour: { gan: string; zhi: string; wuxing: string };
    };
    wuxingDistribution?: { wood: number; fire: number; earth: number; metal: number; water: number };
    xiyongshen?: string[];
    ```
  - 名字列表的 `scores` 保持五维度评分。
- `/api/analyze`
  - 同样返回上述结构化八字数据，保证右侧面板不依赖 generate 阶段的缓存。

### 5.2 前端状态

- `favorites`: `localStorage` 持久化数组，元素为 `{ fullName, givenName, pinyin, score, savedAt }`。
- `history`: `localStorage` 持久化数组，元素为 `{ id, timestamp, request, summary }`。
- `compareList`: session 状态，最多 3 个名字的 `fullName`。
- `formState`: 表单状态，支持从 `/history` 加载。

## 7. 第三方依赖

- 中国省市区数据：使用 `china-area-data` 或内置 JSON，避免引入重型库。
- 农历转换：使用 `lunar-javascript`（轻量，支持公历↔农历）。
- 图标：使用 SVG 内联（不新增图标库依赖，保持 bundle 小）。

## 8. 样式方向

- 主色：`stone` 系（纸质感）+ `amber` 系（印章/高光）+ `emerald`（评分/成功）。
- 表单左列：白底卡片 + 清晰分组标题。
- 列表行：悬停高亮 + 选中态黄色边框。
- 右侧面板：Tab 导航 + 数据可视化 + 评分进度条。
- 全局：`rounded-2xl` 为主，柔和阴影，充足留白。

## 9. 边界与异常

- 未填姓氏时禁用生成并提示。
- LLM 返回格式异常时显示友好错误。
- 收藏/历史读写 localStorage 失败时降级为内存状态并提示。
- 农历转换在极端日期（如 1900 年前）给出兜底提示。
